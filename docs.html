# about

temPHPlate is a template structured in MVC with simplified syntax and easily to use for high-level project development.

this template provides a structure and starting point for creating your application, allowing you to focus on creating something amazing while we sweat the details.

temPHPlate strives to provide a great developer experience, while providing various features such as thorough dependency injection, an expressive database abstraction layer, a view manipulation system, and more

# installation

installation via composer

if your computer already has PHP and Composer installed, you may create a new temPHPlate project by using Composer directly.

            
              $ composer create-project otaviothor/temphplate your-project-name

installation via github

Or, you can start a temPHPlate project cloning from github, just have the git CLI installed.

          
            $ git clone https://github.com/otaviothor/temphplate your-project-name

# configuration

the project configuration for the temPHPlate it is in the file src/Config.php. this configuration file allow you to configure things like your database connection information, your mail server information, as well as various other configuration.

environment configuration

To make this easier, temPHPlate uses the DotEnv PHP library. in a fresh temPHPlate installation, the root directory of your application will contain a called .env.example file that defines many common environment variables. during the installation process, this file will automatically be copied to .env, which is not normally sent to the hosting environment because it is a file with sensible information.

temPHPlate's default .env file contains basics variables ​​that may differ based on whether your application is running locally or on a production web server. these values ​​can be used through the global variables $_ENV['NAME_OF_YOUR_VARIABLE'] and $_SERVER['NAME_OF_YOUR_VARIABLE'], or through function getenv('NAME_OF_YOUR_VARIABLE').

basics variables

variables description
APP_NAME name of your project
APP_DESCRIPTION description of your project
APP_DOMAIN domain that your project will have after being hosted
APP_LOCALE location where your project will be available
APP_ROOT root url of your project
DB_DRIVER database driver used in the project
DB_HOST database host used in the project
DB_PORT database port used in the project
DB_DATABASE database name used in the project
DB_USERNAME database username used in the project
DB_PASSWORD database password used in the project
MAIL_HOST e-mail host used in the project
MAIL_PORT e-mail port used in the project
MAIL_USERNAME e-mail username used in the project
MAIL_PASSWORD e-mail password used in the project
MAIL_FROM_NAME name of the sender's email address used in the project
MAIL_FROM_ADDRESS sender's email address used in the project

# routing

To make this easier, temPHPlate uses the Router PHP library. in a new temPHPlate installation, your application's root directory will contain a file called index.php which will be your project's start file. it should contain all the routes used in your project.

basic structure

            
              $router->verb('uri', 'controller:method', 'route.name');

following the basic structure, we have the variable $router which is an instance of the Router, followed by the function with the verb name of the RESTful pattern, which are GET, POST, PUT, PATCH and DELETE. within the function, we have three parameters. the first one is the uri referring to the route that will be accessed, the second one is the name of the controller next to the name of the method that this route will have to access, the third and last is optional which is the name of the route.

example

            
              $router->get('/', 'WebController:index', 'web.index');
              $router->post('/store', 'WebController:store', 'web.store');
              $router->put('/{id}/update', 'WebController:update', 'web.update');
              $router->patch('/{id}/profile/{photo}', 'WebController:update', 'web.update');
              $router->delete('/{id}', 'WebController:destroy', 'web.destroy');
